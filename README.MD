# Portfolio Manager Admin

A self-hosted admin tool for importing Trade Republic PDFs and Deka CSVs, classifying instruments, and managing savings plans with a deterministic rebalancer workflow that can optionally explain its recommendations via an LLM.

## ⚠️ No Financial Advice

The suggestions generated by this application do not constitute financial advice and should always be critically examined and used with due caution. This tool is a **personal portfolio analysis aid**, not a financial advisor replacement. See [Scope & Limitations](#scope--limitations) below.

## Scope & Limitations

### What this tool does:
- ✅ **Rule-based rebalancing** — Deterministic proposals based on configurable profiles (not LLM-generated)
- ✅ **Portfolio consolidation** — Imports Trade Republic PDFs and Deka CSVs, deduplicates positions
- ✅ **Quality checking** — LLM-assisted research with evidence gates (optional, can be disabled)
- ✅ **Self-hosted** — Your data stays on your infrastructure; no cloud processing
- ✅ **Auditable** — All decisions logged to PostgreSQL; full history retained

### What this tool does NOT do:
- ❌ **Tax-loss harvesting advice** — No tax optimization calculations
- ❌ **Multi-currency support** — EUR only (German market focus)
- ❌ **Real-time monitoring** — Snapshot-based analysis only
- ❌ **Replace a financial advisor** — For complex situations, consult a licensed advisor
- ❌ **Guarantee accuracy** — PDF parsing is imperfect; always verify imported data
- ❌ **Comply with MiFID II** — Personal use only, not regulated financial advice

### Known Limitations:

**PDF Import**
- Supports Trade Republic (stocks/ETFs/funds) and Deka CSV only
- Cannot parse unusual broker formats; requires manual entry
- Decimal separator parsing depends on browser locale (EUR: 1.234,56)

**Rebalancer**
- Optimizes within configured constraints; may not reflect personal preferences
- Variance tolerance (default 3%) is global, not per-layer
- Layer 5 (Unclassified) requires manual classification for better proposals

**Knowledge Base (if enabled)**
- LLM extraction accuracy: ~70-85% first-pass for ETFs/stocks
- May hallucinate citations; quality gates catch most, but spot-checking essential
- Websearch restricted to configured domains; may miss niche instruments

**Assessor**
- Instrument scoring is **heuristic**, not predictive
- Penalty-based model reflects data quality + basic valuation metrics
- Does not account for: macroeconomic conditions, tax implications, personal risk tolerance

### Minimum validation checklist before using:
- [ ] Test with a small portfolio (5-10 instruments) first
- [ ] Verify all imported ISINs match original PDFs
- [ ] Compare rebalancer proposal to manual calculation for one test case
- [ ] Review one complete Knowledge Base dossier extraction (if KB enabled)
- [ ] Keep manual records of all recommendations you act on

## Data Privacy & Security

### Where your data lives:
- **Your portfolio data**: Local PostgreSQL database (self-hosted)
- **LLM queries** (if enabled): Sent to configured LLM provider (OpenAI by default)
  - ⚠️ **Sensitive data filtering**: IBAN, BIC, account IDs, tokens blocked before API call
  - ⚠️ **Domain whitelist**: Knowledge Base websearch limited to your configured domains
- **No telemetry**: This project never contacts external servers or tracks usage
- **No backups to cloud**: All backups are manual and stay on your machine

### Sensitive data handling:
- **Passwords**: Set in .env file. Never stored in in database
- **JWT tokens**: Hashed JTI in database, revocable, 1-hour expiry (default)
- **PDF content**: Extracted contents are stored in local PostgreSQL only
- **LLM prompts**: Scanned for IBAN/email/tokens before API call; filtered requests logged

### Security practices:
- Database runs in Docker container on localhost (not exposed by default)
- All API calls require JWT authentication
- PostgreSQL password defined in `.env` (not in code)
- HTTPS not enabled by default (configure reverse proxy for production)
- Spring Boot logs available for debugging; check for failed LLM calls, SQL errors

### Before production deployment, complete this checklist:
- [ ] **Enable HTTPS** — Use nginx/Caddy reverse proxy
- [ ] **Rotate admin password** — Change `ADMIN_PASS` to strong credential
- [ ] **Verify port binding** — Confirm `ADMIN_SPRING_BIND=127.0.0.1` (not `0.0.0.0`)
- [ ] **Set up PostgreSQL backups** — Encrypted backups to secure storage
- [ ] **Audit LLM API key** — Ensure read-only permissions if possible
- [ ] **Review Docker volumes** — Keep database volumes local, not cloud-synced
- [ ] **Disable Metabase** — Remove from `docker-compose.yml` if not needed
- [ ] **Test JWT secret rotation** — Ensure process is documented

## Features
- Deterministic rebalancer proposals with optional LLM narrative (numeric proposals are rule-based).
- Saving plan projections blend projected gap weights (using effective holdings) with the current savings plan distribution; longer horizons are more conservative (linear 0.15 -> 0.45 over 1-120 months, configurable per profile).
- Rebalancer proposal table shows amounts only (Current EUR, Proposed EUR, Delta EUR) to avoid misleading percentages.
- Assessor view for saving plan adjustments and optional one-time allocations (see Assessor section).

## Architecture

- **PostgreSQL (`db_portfolio`)** stores imported snapshots, instruments, savings plans (`sparplans`), rebalancer history, and layer target configuration.
- **Spring Boot Admin API (`admin_spring`)** exposes the `/api/` surface, runs Liquibase migrations, applies classification rules, manages savings plans, and evaluates the rebalancer.
- **Vue 3 UI (`admin_frontend`)** uses the JWT login flow and lets you upload depot statements, manage reclassification rules, edit savings plans, adjust layer targets, and inspect rebalancer summaries.
- **Assessor** provides a separate assessment view/API (see Assessor section).
- **Metabase (optional)** exposes dashboards; remove or rebind the service in `docker-compose.yml` if you do not need it.
